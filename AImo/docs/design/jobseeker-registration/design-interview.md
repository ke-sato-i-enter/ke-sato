## ヒアリング結果サマリー

### 確認できた事項
1. **会員登録4ステップの詳細**: 基本情報入力のみ（メール・パスワード → 基本プロフィール → 質問3つ → 完了）
2. **認証方式**: NextAuth.js Credentials Provider（メール+パスワード）
3. **データモデル**: 求職者フルプロフィール（職歴、スキル、希望条件、自己PR、写真）
4. **Phase 1スコープ**: 会員登録、ログイン/ログアウト、プロフィール編集、写真アップロード
5. **新規要件**: 質問3つでスキル・経歴を登録する対話形式プロフィール作成

### 設計方針の決定事項
1. **アーキテクチャ**: Next.js フルスタック（App Router + API Routes）
2. **認証**: NextAuth.js v5 Credentials Provider + JWT
3. **データベース**: Prisma + SQLite（開発）→ PostgreSQL（本番）
4. **ファイルストレージ**: Cloudflare R2 / AWS S3（写真アップロード）
5. **UI/UX**: 
   - 4ステップの段階的登録フロー
   - 対話型質問コンポーネント（スキル・経歴入力）
   - モバイルファーストデザイン（スマホ最適化）
6. **セキュリティ**: 
   - パスワードハッシュ化（bcrypt）
   - JWT セッション
   - Zod サーバーサイドバリデーション
   - HTTPS（Vercel自動対応）

### 残課題
1. **質問3つの具体的な内容**: 実装時に決定（例: 「これまでの職歴を教えてください」「得意なスキルは何ですか」「希望する職種は何ですか」）
2. **写真アップロードの詳細仕様**: ファイルサイズ上限（5MB推奨）、対応形式（jpg/png）、リサイズ処理の要否
3. **バリデーションルール詳細**: パスワード強度（最小8文字推奨）、メール形式、必須項目の具体化
4. **エラーメッセージの文言**: UX観点での具体的なメッセージ内容（実装時に決定）
